Recently I dusted off my web design business. Since I’ve become a pastor, I’ve had the privilege of not needing to do much outside work, but circumstances have made it plain to me that I need some outside income, and for me, outside income has always meant making web sites.

I talk to lots of people interested in making web sites: small business owners trying to do it themselves, the guy at the church who is unlucky enough to be given responsibility over the church website, and young “I want to be a designer when I grow up” types. Whatever their reasons, I think that’s really cool.

If you are one of those people, I want you to be able to gain from my experience. So here is:

Five things every aspiring web designer needs:

1. Realistic expectations


When I started making websites a decade ago, I thought you could do the whole thing in photoshop. I went into photoshop and laid out an awesome looking website, used the photoshop button tools (which i don’t even think exist anymore) to make buttons and links, and uploaded a massive (for then) webpage that worked correctly on almost 0 browsers.

Having accomplished this - I set out to start a web design business. My first paying web site job was for a seasoned graphic designer. I was really just pretending at that point.

I was foolish. I was naive.

Ten years and who knows how many websites later, I am less, not more, confident in my design and coding ability than I was then. Web design is a lot of work, and there is a lot to know.

On top of basic graphic design skills. You really need to know:

  • HTML & CSS
  • A little about web standards
  • A little about information architecture
  • A little about user experience and interaction design
  • How to use a web host
  • How to use FTP
  • What responsive design is
  • How to make mobile web sites
  • How to use Wordpress or another Content Management System
  • The list could go on…


So if you want to learn web design - go for it. But have realistic expectations getting into it. Expect to spend a lot of time over the course of years teaching yourself the things above. Expect long hours where you feel like banging your head against a desk and throwing your monitor out the window. Realize you are taking on a massive and rapidly transforming subject that you probably will never stop learning about.

If you have that mindset from the beginning, you’ll be halfway there.

2. Learning resources:


I think every beginning web designer should have at least these learning resources:

A Primer


In my opinion, the very best book for beginning web designers is “Build Your Own Website From Scratch the Right Way Using HTML and CSS” by Ian Lloyd. It’s been around forever, and has been updated several times, but if you read it carefully it will get you a very good basic introduction to web design and development.

The Boagworld Podcast


Boagworld is a British Web Design podcast that’s been going on forever. Its entertaining, and informative, and if you listen to it…

  • you’ll stay up to date with what’s going on in the web design world.
  • you’ll learn about new trends.
  • you’ll learn the sources the pro’s turn to.
  • you’ll learn about best practices.


You will get a better education about web design and development by just listening to that podcast and researching the stuff they talk about than you would ever get in a college course on web design.

Learning websites


On top of Boagworld, I recommend you read these websites:

  • Sitepoint
  • Smashing Magazine
  • A List Apart

Some design galleries


You are going to need to know what good web design looks like, and there is probably no better way to learn than looking at a lot of great websites. For this purpose, web design galleries are an invaluable resource. Here are some of my favorites:

  • CSSelite
  • CSSremix
  • CSSmania
  • Smashing Magazine

Books


All of these things are going to fill you with a desire to learn more about specific areas of web design. So expect to buy a lot of books. These books can get pretty expensive, and there are a lot of them out there. Three web design focused publishers that usually put out great books are:

  • Sitepoint
  • Smashing
  • ABookApart


I told you this wasn’t going to be easy, but this is the cost of being a good web designer.

The third thing you’ll need, after tenacity and some learning resources is…

3. A testing environment


You are going to need a way to test sites locally on your computer. The easiest way to do this on a Mac is with a free program called MAMP, on Windows, you can use a free program called WAMP.

[caption id="attachment_1887" align="alignnone" width="300"] MAMP running on my computer[/caption]

These programs will put a folder on your computer that basically acts like a web host. You can set up wordpress or another CMS on it and play around as much as you want without other people seeing your work.

Thankfully, this step doesn’t cost money, and is fairly easy to set up.

The fourth thing you will need is…

4. Layout Software


The gold standard layout software for web design is Photoshop. That’s what I used for about 10 years. But I don’t use it anymore.

There are two reasons for that:

  • Cost - Adobe Photoshop costs $10 a month (or you can get all of adobe’s tools for $50 a month.) That’s really not that bad, but if you stop paying the subscription, you lose the ability to edit all of the files you’ve created.
  • Usability - Adobe Photoshop, as it’s name suggests, is made for editing photos. It was never intended to be used for laying out websites. So it’s kind of a cumbersome solution.


So what do you use if you don’t use photoshop? An even better solution, in my opinion, also happens to be much cheaper. If you have a Mac, you can use the program that I now use, called Sketch (by Bohemian Coding.) It’s very simple, easy to use software that is built for interface design (think web design) and it’s very, very powerful. It also has the added benefit of costing only $79.

[caption id="attachment_1888" align="alignnone" width="300"] A mockup I'm working on in sketch.[/caption]

 

I can lay something out in Sketch in a small fraction of the time it would take me to do the same thing in Photoshop and the finished product is much more web friendly. Plus, Sketch runs much, much faster on my aging MacBook than Photoshop did.

You can download a free 14 day trial of Sketch here.

5. Coding Software.


You are going to need coding software. O.k., you don’t need this. You can actually get by with the textedit app built into your mac or the notepad app built into your PC. But you are going to want coding software.

A word of caution here - stay far away from WYSIWYG programs (Like dreamweaver, flux or rapidweaver) and learn to code. If you really want to learn Web Design, there is no way around this.

In my opinion, the very best coding program is called Espresso. (Unfortunately, like Sketch, it’s Mac Only.) What makes Espresso the best? It does what all of the similar programs do: it highlights your code, helps autocomplete tags, and helps you keep your code organized but where Espresso shines is in the way it handles CSS. There used to be a product called CSSedit that was the best way to style a website. CSSedit is no more, but the entire product has been brought into Espresso and updated.

[caption id="attachment_1889" align="alignnone" width="300"] A CSS file in Espresso. Notice the inspector on the right - very powerful.[/caption]

You may have no idea what that means now, but let me tell you, when you start getting into serious CSS, you’ll appreciate it. You can try and buy espresso here.

A free alternative to Espresso that also runs on Windows is Brackets. It’s not as feature rich as Espresso, but it is a great code editor and, well, it’s free. You can find out about Brackets here.

The Sixth thing you’ll need is…

6. An FTP Program.


When you finally have a web site done, you’ll need an FTP program to get it onto a web host. FTP stands for File Transfer Protocol and is a secure way to transfer files to a web host. A great free (they ask for donations) program is Cyberduck. This is available on both the Mac and Windows.

Other great FTP programs include Forklift (what I use), Transmit, and Filezilla.

7. A project to start on.


Once you’ve read some books, set up a testing environment, and learned some software. You’ll need a project.

If you really want to get into web design, your best first project will probably be a personal web site for yourself. Because it’s your web site, it can be terrible, it can evolve quickly, and it can change often. So set up a personal site and then work on improving it.

As your confidence builds, then start taking on other sites. Start small and simple, and work your way up to complex.

If you are like me, you’ll have found something you really enjoy. If you need any help, please drop me a line and I’ll do what I can.
